Вывод массива с помощью метода PrintGood является наиболее предпочительным относительно метода PrintBad по следующим причинам:
1. Более информативен в случае вывода на экран массива и другой информации в одну строку методом WriteLine(Например [Здесь находится массив в текстовом виде] здесь прочая информация).
2. В случае если массив не будет содержать никакой информации, это будет наглядно показано пустым содержимым внутри квадратных скобок, в то время как наличие просто пустой строки может говорить о некорректности написанного кода.

При этом, в любом случае на выходе получается string, что в дальнейшем затрудняет обработку указанной информации как массива (потребуется перевод string в int[] array или string[] array )
